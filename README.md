    <article class="markdown-body entry-content container-lg" itemprop="text"><h1><a id="user-content-ebike-handlebar-remote-controller" class="anchor" aria-hidden="true" href="#ebike-handlebar-remote-controller"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>EBike Handlebar Remote Controller</h1>
<p>This project implements a wireless remote button for EBikes and/or GPS units like the Garmin Edge. You can buy on the market a similar product to this remote - see the <a href="https://buy.garmin.com/en-US/US/p/545795/pn/010-12094-30" rel="nofollow">Garmin EBike Remote for about 80‚Ç¨</a>.
This wireless remote button implements two ANT+ profiles:</p>
<ol>
<li>the ANT+ CTRL profile allowing it to control the page view on compatible bike computers - see the user manual of your device to see if it supports this profile. The remote will pair with any device that supports this profile.</li>
<li>The ANT+ LEV (Light Electric Vehicle) profile allowing the remote to control various features of the eBike including  changing the assist level. The remote will work with any eBike that support the ANT+ LEV profile. Of course, the upcoming TSDZ2 wireless eBike controller will support this profile.
see: <a href="https://github.com/OpenSource-EBike-firmware/TSDZ2_wireless">wireless EBike TSDZ2</a>.</li>
</ol>
<p>For questions, please post on <a href="https://endless-sphere.com/forums/viewtopic.php?f=28&amp;t=106346" rel="nofollow">this forum message</a>.</p>
<p>See controlling a Garmin 1030 bike computer for assist levels and page control using a simulated ANT+ LEV Ebike in this video:</p>
<p><a href="https://www.youtube.com/watch?v=s7URIMVzcwc" rel="nofollow"><img src="https://camo.githubusercontent.com/6765537f5f61c37776316ddfa830e6ce0a90eba8db3c718a00f59520caf26f03/68747470733a2f2f696d672e796f75747562652e636f6d2f76692f73375552494d567a6377632f687164656661756c742e6a7067" alt="video" data-canonical-src="https://img.youtube.com/vi/s7URIMVzcwc/hqdefault.jpg" style="max-width:100%;"></a></p>
<p>See changing the ANT+ LEV Device Number (to connect to only one specific eBike) using the Nordic nRF Connect app in this video:</p>
<p><a href="https://www.youtube.com/watch?v=pvqh7uVm5ww" rel="nofollow"><img src="https://camo.githubusercontent.com/e7f73a10831ea82b52ea73d6dc3c5abf02d44ea9c8d7ddb1358ed8939ba8509e/68747470733a2f2f696d672e796f75747562652e636f6d2f76692f707671683775566d3577772f687164656661756c742e6a7067" alt="video" data-canonical-src="https://img.youtube.com/vi/pvqh7uVm5ww/hqdefault.jpg" style="max-width:100%;"></a></p>

<h2><a id="user-content-current-state" class="anchor" aria-hidden="true" href="#current-state"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Current state</h2>
<p>2020.11.04:
The remote is using the inexpensive VLCD5 handle bar remote keypad for control.</p>
<ul>
<li>
<p>The firmware works well as an Edge Remote to change page views on the bike computer (tested on Garmin Edge devices).</p>
</li>
<li>
<p>The firmware works well as a LEV remote control to change assist levels on an e-bike.</p>
</li>
<li>
<p>The remote will go into ultra low power mode if it fails to connect to a device within ~30 seconds, or if no buttons are pressed for 1 hour. The system wakes up from ultra low power when the user presses any button.</p>
</li>
<li>
<p>OPERATION:</p>
</li>
<li>
<p>Short Press the [MENU] button to switch pages.</p>
</li>
<li>
<p>Short press the [PLUS] button to increase the motor assist level</p>
</li>
<li>
<p>Short press the [MINUS] button to decrease the motor assist level</p>
</li>
<li>
<p>Long press the [PLUS] button to enter bluetooth mode to allow the ANT Device number to be set for the ebike you would like to control. (Device number '0' will pair with any ebike device number)</p>
</li>
<li>
<p>Long press the [Minus] button to exit bluetooth mode to save power. (Bluetooth will automatically turn off after:</p>
</li>
<li>
<p>10 minutes if you left bluetooth running</p>
</li>
<li>
<p>after you enter a new ANT+ LEV Device Number</p>
</li>
<li>
<p>What is left to be done:</p>
<ul>
<li>Bluetooth enabled bootloader to allow for wireless firmware updates</li>
<li>Evaluate sending the brake sensors signal wireless</li>
<li>Planned feature: Long press the [POWER] button to turn off the TSDZ2 motor</li>
<li>Other ANT+ LEV button assignments (ie: Lights on/off Turn signals, etc)</li>
<li>no battery voltage measurement on the hardware, firmware and so it is not reported to master / Edge</li>
<li>Design a suitable spacer for 3D printing that can reuse the VLCD5 keypad and handlebar connector (basically, the top and bottom portions of the keypad)  Inital design attempts are located in the 3d_design folder. Please contribute if you can!</li>
</ul>
</li>
</ul>
<p><strong>Components you will need:</strong></p>
<ul>
<li><a href="https://makerdiary.com/products/nrf52840-mdk-usb-dongle" rel="nofollow">nRF52840 MDK USB Dongle</a>: (costs 11‚Ç¨) this is this wireless board</li>
<li><a href="https://en.wikipedia.org/wiki/Button_cell" rel="nofollow">CRC2032 button cell</a>: (costs 1‚Ç¨)</li>
<li><a href="https://www.aliexpress.com/wholesale?catId=0&amp;initiative_id=SB_20200828081711&amp;origin=y&amp;SearchText=LCD+controller+of+VLCD5+display+for+TSDZ2+electric" rel="nofollow">VLCD5 handle bar remote keypad</a>: (costs 6‚Ç¨) can be any keypad that works in the similar way</li>
</ul>
<h2><a id="user-content-schematic" class="anchor" aria-hidden="true" href="#schematic"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M7.775 3.275a.75.75 0 001.06 1.06l1.25-1.25a2 2 0 112.83 2.83l-2.5 2.5a2 2 0 01-2.83 0 .75.75 0 00-1.06 1.06 3.5 3.5 0 004.95 0l2.5-2.5a3.5 3.5 0 00-4.95-4.95l-1.25 1.25zm-4.69 9.64a2 2 0 010-2.83l2.5-2.5a2 2 0 012.83 0 .75.75 0 001.06-1.06 3.5 3.5 0 00-4.95 0l-2.5 2.5a3.5 3.5 0 004.95 4.95l1.25-1.25a.75.75 0 00-1.06-1.06l-1.25 1.25a2 2 0 01-2.83 0z"></path></svg></a>Schematic</h2>
<p><a target="_blank" rel="noopener noreferrer" href="/OpenSource-EBike-firmware/ebike_wireless_remote/blob/master/hardware/ebike_remote_wireless-v1.png"><img src="/OpenSource-EBike-firmware/ebike_wireless_remote/raw/master/hardware/ebike_remote_wireless-v1.png" alt="schematic" style="max-width:100%;"></a></p>
<p><strong>Explanation:</strong></p>
<ul>
<li>The CRC2032 button cell powers up the wireless board. Connect the cell positive to the pin 10 / VIN and the negative to the pin 8 / GND.</li>
<li>VLCD5 handle bar remote keypad has 4 buttons and 5 wires (1 wire for each button and 1 common wire to all buttons). Connect the wires directly to the wireless board.</li>
<li>For flashing the firmware, that you will need to do only one time, solder the 4 wires to the STLinkV2 (I use a cheap 1‚Ç¨ STLinkV2 clone that I bought on EBay).</li>
</ul>
